## Aula 3.3 — Framework próprio (não importe pronto) (5 min)

**Tema:** A armadilha de adotar framework pronto (BMAD, AIOX, Speckit etc) sem entender o que ele resolve. Por que o instrutor passou a desenvolver harness próprio em vez de seguir framework externo. Como começar simples: 1 documento de regras + 1 fluxo + 1 critério de aceite — evoluir conforme atrito real aparece.

**A tese:** modelo é commodity. Framework também. O que diferencia é o **harness que você constrói pro seu caso**. Importar framework de outro é tão arriscado quanto importar arquitetura de outro produto.

**Regra prática:** começa sem framework. Quando aparecer fricção repetida, escreve a regra que evita aquilo. Ao final do 30d você tem 5-10 regras que viraram seu próprio framework — sob medida pro seu projeto.

**🏆 Troféu:** Você tem 3 regras escritas no CLAUDE.md que vieram de problema real seu, não de framework copiado.

## Aula 3.4 — Arquitetura (5 min)

**Tema:** Desenhar o fluxo Input → LLM → Processamento → Output. Quando entra cache, quando entra fila, quando entra DB. Aplicar os princípios de `guias/05-system-design-vibe-coder.md`.

**🏆 Troféu:** Você desenhou o sistema do seu projeto no Excalidraw com componentes nomeados.

---

## Aula 3.5 — Stories (5 min)

**Tema:** Quebrar o PRD em pedaços executáveis (stories/issues). Cada story pequena o suficiente pra terminar numa sessão com Claude Code.

**🏆 Troféu:** Você tem ≥ 5 stories prontas pra começar a executar.
