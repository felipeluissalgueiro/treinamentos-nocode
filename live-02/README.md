# Live 02 — De Passageiro a Dono

> O que o Vibe Coding está escondendo de você

**Duração:** 2 horas | **Data:** Agosto/2026

---

## O que você vai aprender

1. Por que vibe coding é uma bomba relógio — de custo, de lock-in e de dívida técnica
2. Como escolher o modelo de IA certo para cada tarefa (e quanto custa errar)
3. O que é uma RFC e como ela muda completamente a forma de trabalhar com IA
4. Os 3 princípios que a IA quebra por padrão — DRY, ETC e Ortogonalidade
5. Como quality gates automatizam o que o code review humano não cobre
6. Qual é o piso mínimo de conhecimento técnico para parar de ser passageiro

---

## O que você vai ser capaz de fazer depois dessa live

- Identificar quando a IA está gerando código acoplado, repetido ou sem componentização
- Escrever uma RFC básica antes de mandar um agente codar
- Configurar um quality gate simples que force a IA a modularizar
- Nomear o próximo passo — saber o que estudar e em que ordem para subir de nível

## Como estudar este material com IA

Cole este prompt no seu modelo de IA:

```
Você vai me ajudar a estudar a Live 02 do curso NoCode Startup.
Leia o arquivo conteudo.md e me faça um quiz de 5 perguntas sobre os conceitos,
uma por vez. Corrija minhas respostas e explique onde errei.
```

Ou, para aplicar ao seu produto:

```
Você vai me ajudar a aplicar o que aprendi na Live 02.
Tenho um produto [descreva seu produto].
Me ajude a escrever uma RFC para a feature [descreva a feature],
usando o template do arquivo demos/demo-rfc.md.
```

---

## Tarefa de fixação

**Antes da próxima live:**

Escreva uma RFC para uma feature do seu produto usando o template em `demos/demo-rfc.md`.

Estrutura mínima:
- Contexto (por que essa feature existe)
- Entradas (o que entra)
- Saídas (o que sai)
- Regra de negócio principal
- O que está fora do escopo

Compartilhe no grupo se quiser feedback.
